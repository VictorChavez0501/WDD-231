Lumina Seed
Project Site Plan — Individual WDD 231 assignment

Overview

This document is the planning blueprint for Lumina Seed — a simple portfolio / product site for an app concept centered on a seed that evolves into light-based growth (branding concept). The plan covers site name rationale, purpose, user scenarios, color & typography, simple wireframes for mobile & desktop, CSS notes, and testing checklist.

1. Site Name

Lumina Seed

Reason: evokes the brand idea — a small seed that evolves and emits light. Suitable for an app, portfolio, or small product focused on growth, mindfulness, or creativity.

Optional domain: lumina-seed.com (representative)

2. Site Purpose

The site will present the Lumina concept and app features, showcase screenshots and the app's evolution story, and provide a call-to-action for downloads or contact. It will include:

  • Home / hero with brand statement and CTA
  • Features / How it works section
  • Gallery / screenshots
  • About / story
  • Contact / download links

3. Scenarios (target visitor questions)

At least two scenarios to guide content:

  • Visitor: "What does the Lumina app do and how will it help me?" — Answer with clear features and short demo GIF or screenshots.
  • Visitor: "How do I download or contact the developer?" — Provide download links and a contact form / email.

4. Color Schema (applied on this document)

The selected palette is applied on this page. Colors and usage:

#0F4C81Headings, nav
#F7C948Buttons, accents
  • --accent-1 (#0F4C81): headings, links, primary nav.
  • --accent-2 (#F7C948): CTA, highlights, card accents, buttons.
  • --bg / --card: page and card backgrounds for high contrast and readability.

Note: The assignment required using the chosen color scheme on the plan itself — this page uses those exact colors.

5. Typography

Selected fonts (1–3 fonts):

Inter

Used for body text, UI, captions.

Playfair Display

Used for brand headings and large titles.

Usage map:

  • Headings: Playfair Display (bold) — hero, section titles.
  • Body: Inter regular (300–400) — paragraphs, lists.

CSS

Supporting CSS notes:

  • Use a single stylesheet (e.g. styles.css) or internal styles for this plan document. This plan uses styles embedded in the page to ensure the selected color scheme and typography are applied.
  • Responsive breakpoints: mobile-first then expand at 640px and 1024px.
  • Accessibility: use semantic HTML (header, nav, main, footer), proper headings, alt text for images.

Testing

Planned self-checks and validation tools:

  • HTML validation: W3C Markup Validation.
  • CSS validation: W3C CSS Validation service.
  • Color contrast: check headings and background using a contrast tool; ensure AA at minimum for body text.
  • Accessibility: run Lighthouse accessibility audit and manual keyboard navigation test.
  • Performance & SEO: check Lighthouse best-practices and SEO scores; optimize images and reduce large assets.
  • Cross-browser testing: Chrome, Firefox, Edge, and mobile devices (Android & iOS).

Self-evaluation: After publishing, load the GitHub Pages URL and run Lighthouse in Chrome DevTools — fix any critical accessibility or performance issues before final submission.
